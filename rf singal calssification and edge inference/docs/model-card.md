# Model Card: RF Spectrogram CNN

## Summary

RF Spectrogram CNN is a synthetic-data baseline for receive-only RF edge signal awareness. It classifies compact complex-baseband I/Q windows into six families:

- `bpsk`
- `qpsk`
- `fsk`
- `ofdm`
- `fm`
- `noise`

The model is intended for portfolio demonstration of edge AI engineering practices: synthetic RF data generation, model export, runtime benchmarking, calibration, scenario testing, and evidence-backed deployment decisions.

## Safety Scope

This model is receive-only and defensive.

- It does not transmit RF.
- It does not jam, spoof, interfere, or decode protected communications.
- It does not perform target selection, guidance, or engagement.
- It should only be evaluated with synthetic data or lawful receive-only lab/public RF captures.

## Architecture

- Input: one complex I/Q window.
- Preprocessing: compact 5-channel RF spectrogram tensor.
- Channels: log power, normalized STFT real part, normalized STFT imaginary part, phase moments, phase-transition histogram.
- Inference model: small fixed-filter convolutional feature extractor plus ridge-fitted dense head.
- Export format: ONNX.
- Accepted deployment artifact: FP32 ONNX.

## Training Data

The current model is trained only on synthetic RF windows generated by the project. See `dataset-card.md` for details.

No real SDR recordings are used for training in this version. Public SatNOGS IQ chunks are used only for domain-shift evaluation.

## Intended Use

Use this version to demonstrate:

- Edge RF classification pipeline design.
- Synthetic RF data generation and scenario stress testing.
- ONNX export and runtime comparison.
- Quantization evaluation and rejection criteria.
- Dashboard/event-stream integration for operator-facing telemetry.

Do not use this version as an operational RF classifier. It has not been validated against real hardware captures, real channel impairments, or mission-specific RF environments.

## Quantization Decision

Static INT8 QDQ quantization reduced the ONNX artifact from 152,465 bytes to 42,080 bytes. Accuracy dropped from 0.8625 to 0.2625 on the nominal benchmark. This candidate is rejected for deployment.

Accepted artifact: `rf_spectrogram_cnn.onnx`

Rejected artifact: `rf_spectrogram_cnn.int8.onnx`

Reason: accuracy regression is too large for the latency and size benefit.

## Limitations

- Synthetic-only data.
- Simplified channel model.
- Low confidence scores indicate calibration is not yet production quality.
- Accuracy degrades materially under low SNR, dropouts, and clock drift.
- No hardware-in-the-loop validation.
- No real-time memory, thermal, or power profiling yet.

## Next Improvements

1. Train a learnable CNN or compact transformer using PyTorch, then export to ONNX.
2. Add quantization-aware training or architecture changes designed for INT8.
3. Improve simulation realism: multipath, Doppler, adjacent-channel interference, burst timing, and front-end clipping.
4. Add calibration metrics such as ECE and reliability diagrams.
5. Add receive-only SDR capture support when hardware is available.
